# 双语界面设计

## 目标

为现有 PDF 图片提取页面提供中英文双语支持，默认跟随浏览器语言，并允许用户通过显式切换入口在中文和英文之间切换。

## 设计结论

### 1. 语言选择策略

- 默认读取请求头里的 `Accept-Language`
- 当前仅在 `zh` 和 `en` 之间选择
- 如果 URL 中带有 `?lang=zh` 或 `?lang=en`，则优先使用 URL 参数

### 2. 页面切换入口

- 在页面顶部加入轻量的 `中文 / EN` 切换入口
- 切换通过 URL 参数实现，保证语言状态可分享、可刷新、可回退

### 3. 文案组织方式

- 在 Python 侧维护一份轻量翻译字典
- 模板渲染时传入当前语言和对应的翻译对象
- 模板静态文案统一从翻译对象中读取
- 前端脚本中的状态提示、错误提示、弹窗文案也从注入的翻译对象中读取，不再硬编码中文

### 4. 实现边界

本次仅覆盖：

- 首页页面文案
- 上传区状态提示
- 错误卡片
- 结果弹窗
- 语言切换入口

本次不覆盖：

- README 双语化
- API 错误 `detail` 的多语言返回
- 用户语言偏好持久化
- 第三方 i18n 依赖
